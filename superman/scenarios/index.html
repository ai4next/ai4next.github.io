Use Cases

典型使用场景

01

个人开发助理：在本地仓库中理解代码、执行命令、修改文件、追踪 revision，并把项目习惯沉淀为长期 SOP。

02

团队知识自动化：把日常问题、排障流程、代码规范、部署步骤沉淀到扁平文件记忆中，持续改善团队 Agent 的工作方式。

03

多平台智能入口：通过 IM server 把 Superman 接入企业常用聊天平台，让任务从消息入口进入，但在后台使用统一会话和工具体系处理。

04

可治理的 Agent 实验平台：使用 Hook、audit log、MCP 和 Skill 系统快速组合工具链，同时保留拦截、审计和回放能力。

场景展开

代码库维护

Superman 可以在项目目录中读取文件、执行测试、定位失败、修改代码，并把文件修订纳入会话历史。适合处理“小到修 bug，大到理解模块边界”的工程任务。

  • 阅读 README、配置、测试和源码。
  • 运行构建、测试、格式化命令。
  • 用 patch/write 修改文件并保留 revision。

运维与排障

通过命令执行、日志读取、会话保存和 SOP 沉淀，Superman 可以把一次排障过程变成团队后续可复用的操作经验。Hook 还能把关键操作接入审批或通知。

  • 读取日志与配置。
  • 执行健康检查脚本。
  • 把稳定流程沉淀到 L2 SOP。

专家协作

当任务需要不同领域判断时，主 Agent 可以把子任务委托给专家。每个专家有自己的 soul、记忆和会话，适合长期培养代码审查、数据分析、文档整理、自动化测试等角色。

  • 主 Agent 负责拆解和整合。
  • 专家负责独立分析。
  • 专家经验不污染主 Agent 记忆。

组织聊天入口

通过 sm im serve，团队可以从飞书、Slack、企业微信、QQ、Discord 等平台发起任务。消息入口只负责触发，后台仍使用 Superman 的会话、工具和审计体系。

  • 适合群内问答和自动处理。
  • 可配置 allow_from 控制来源。
  • 可按平台选择进度展示方式。
落地建议：先从单人本地代码库任务开始，确认配置、工具和会话历史符合预期，再接入专家、Hook 和 IM 平台。
