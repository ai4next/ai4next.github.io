扩展体系

MCP Server

通过 stdin/stdout transport 接入任意 MCP-compatible server，并可按工具名过滤。

Skill 系统

从多个 skill path 自动加载 SKILL.md，兼容 Claude Code 风格的技能目录。

Hook 系统

在 before/after run、tool、model、agent、用户消息和错误事件中执行外部脚本。

Plugin 系统

统一处理 run/model/tool 日志与 session reaper 等后台能力。

MCP：接入外部工具服务

MCP server 适合把已有工具、系统 API 或第三方服务挂到 Superman 上。配置项支持 server name、command、args、enabled 和 tools 过滤，空 tools 列表表示暴露全部工具。

mcp:
  servers:
    - name: filesystem
      enabled: true
      command: npx
      args: [-y, @modelcontextprotocol/server-filesystem, /tmp]
      tools: []

Skill：把技能说明放进文件系统

Skill 系统支持多个路径，适合把代码审查、数据清洗、文档生成、测试策略等可复用工作方法沉淀为目录。每个技能以 SKILL.md 为入口，可带 references 等辅助资料。

skills:
  enabled: true
  paths:
    - ${HOME}/.sm/skills
    - ./skills

Hook：在生命周期中插入组织规则

Hook 脚本通过 stdin 接收 JSON context，通过 stdout 返回 JSON。它适合做工具调用前审批、模型错误告警、运行后归档、用户消息记录等动作。

事件用途
before_run / after_run任务开始前后执行初始化、记录或清理。
before_tool / after_tool工具调用前后做审批、参数检查、审计或通知。
before_model / after_model模型调用前后记录请求上下文、统计耗时或接入网关策略。
on_model_error / on_tool_error错误发生时告警、降级或记录排障信息。

即时通信扩展

IM server 是另一类扩展入口。Superman 的配置示例覆盖 Telegram、飞书/Lark、企业微信、微信个人号、QQ、QQ 官方机器人、钉钉、Slack、Discord、LINE 和微博。不同平台可配置来源限制、群聊回复策略、线程隔离、进度样式等选项。

扩展原则：外部能力可以接进来，但最好保持配置可见、工具可列出、事件可审计、权限可限制。
