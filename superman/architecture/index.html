Architecture

核心架构

Superman 的架构重点是“群体”和“边界”：主 Agent 协调工具与专家，专家拥有独立记忆与会话，进化系统只在明确目录中改写长期能力。

Superman 主 Agent
工具层
exec / read / write / patch / ask
专家层
delegate + 独立专家记忆
扩展层
MCP / Skills / Hooks
会话层
SQLite + compact logs + snapshots
记忆层
L1 facts + L2 SOP
进化层
agent evolver + meta evolver

主 Agent 记忆、专家记忆、专家 soul、evolver 记忆、会话、审计日志分别落在不同目录和存储中。系统可以学习，但学习结果必须进入明确的位置，避免自我改进变成不可控漂移。

CLI 与 Runtime

internal/cli 负责 Cobra 命令入口，包括 init、run、reflect、im、sessions、runtime、toolsets 等命令。internal/runtime 负责事件驱动执行、审计日志和运行循环，让交互式界面、单次执行和 IM server 能复用同一套核心运行逻辑。

Agent 工厂

internal/agent 构建主 Agent、专家 Agent 和 evolver Agent，注入模型、工具、记忆、SOP、上下文和委托能力。它是把配置转换成实际可运行 Agent 的核心位置。

Session 与 Store

internal/session 提供持久 SessionService，负责上下文构建、压缩、文件 revision tracking、导入导出和引用关系。底层 internal/store 使用 SQLite/GORM 保存会话、消息和文件元数据。

Memory 与 Expert

internal/memory 使用扁平文件组织 L1 facts 和 L2 SOP。internal/expert 用目录注册专家，专家名称来自目录名，soul.md 是该专家的系统提示。

运行时数据边界

Superman 默认 workspace 是 ~/.sm/。运行时目录不仅用于保存数据，也表达责任边界：主 Agent 的记忆不和专家记忆混在一起，evolver 的记忆不和普通会话混在一起，审计事件也不伪装成长期知识。

memory/l1.toml：主 Agent 的全局事实。
memory/l2/*.md：可复用 SOP、流程、约定和操作经验。
experts/{expert_name}/soul.md：专家的系统提示词和职责定义。
sessions/：精简会话日志与文件 revision snapshots。
runtime/events.jsonl：工具调用、文本增量、错误和进化事件。
evolution/：agent evolver 与 meta evolver 的独立运行根目录。
架构重点：Superman 倾向用小文件、显式目录和可检查 diff 管理长期状态，而不是把所有内容隐藏在不可见上下文里。
