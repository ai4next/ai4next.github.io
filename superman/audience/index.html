Audience

适合谁使用

希望在本地代码库中使用可执行、可写文件的 AI Agent 的开发者。
需要把团队流程和经验持续沉淀为 Agent 记忆的工程团队。
正在探索多 Agent、专家协同、Agent 自进化和治理边界的研究者。
想把 AI 能力接入飞书、Slack、企业微信、QQ、Discord 等聊天平台的自动化团队。

使用者画像

个人开发者

如果你希望 Agent 能进入本地项目目录，理解代码、执行命令、修改文件并保留历史，Superman 可以作为日常开发助理使用。

工程团队

如果团队有大量重复排障、代码规范、发布流程和项目约定，Superman 的 L1/L2 记忆和专家体系适合持续沉淀这些经验。

平台团队

如果你需要把 Agent 接入聊天平台、内部工具、审批流程和审计系统，MCP、IM server、Hook 和 runtime events 提供了较完整的扩展入口。

Agent 研究者

如果你关注多 Agent 协作、长期记忆、分层自进化和边界治理，Superman 提供了一个可运行、可检查、文件系统优先的实验基座。

不适合的情况

如果你只需要一个网页聊天机器人，或者完全不希望 Agent 接触本地文件、命令执行和长期状态，Superman 可能不是最轻的选择。它的价值主要体现在长期协作、工具执行、文件系统记忆和可治理的自治能力上。

推荐采用路径

  1. 先本地单人使用：配置模型，运行 TUI 和 sm run，确认基础能力可靠。
  2. 再接入项目流程：让它处理 README 总结、测试失败分析、脚本执行、文档更新等低风险任务。
  3. 然后沉淀记忆和专家：把稳定经验放入 L2 SOP，按领域建立专家目录。
  4. 最后接入组织入口：通过 IM server、Hook、MCP 和审计日志接入团队工作流。

把一次性对话，变成持续进化的工作系统。
